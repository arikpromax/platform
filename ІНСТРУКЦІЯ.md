# 📖 Посібник власника платформи arawebsite

Адмінка: **https://platform-one-bay.vercel.app**
База: Supabase, проєкт **platform** (supabase.com → Dashboard → platform)
Код: **github.com/arikpromax/platform** (оновлюється командою `git push` — Vercel передеплоює сам)

---

## A. Як дати клієнту доступ до його сайту (акаунт власника)

**Крок 1. Створити користувача.**
Supabase → проєкт **platform** → **Authentication → Users → Add user → Create new user**:
- email клієнта (справжній)
- тимчасовий пароль — придумати і **записати собі**
- ✅ галочка **Auto Confirm User** (обов'язково!)

**Крок 2. Дізнатися номер сайту** (якщо не пам'ятаєте).
**SQL Editor** → виконати:
```sql
select id, slug, name from public.sites;
```
У відповіді буде табличка: номер, коротке ім'я, назва. (SUSHI BOOM = **1**)

**Крок 3. Прив'язати користувача до сайту.**
**SQL Editor** → підставити email і номер сайту → **Run**:
```sql
insert into public.profiles (user_id, site_id, role)
select id, НОМЕР_САЙТУ, 'owner' from auth.users where email = 'EMAIL_КЛІЄНТА';
```
Результат «Success, 1 row» = готово. Якщо «0 rows» — користувача з таким email нема (див. Крок 1).

**Крок 4. Передати клієнту** (особисто, не в публічний чат):
- адресу адмінки: https://platform-one-bay.vercel.app
- його email і тимчасовий пароль

**Перевірка:** відкрити вікно інкогніто (`Ctrl+Shift+N`) → увійти акаунтом клієнта.
Має відкритися ОДРАЗУ його сайт (без списку всіх сайтів) — і тільки його.

> ⚠️ Якщо користувача видалили і створили заново — Крок 3 треба виконати ще раз.

## B. Як додати НОВИЙ сайт у платформу

Поки що нові сайти додаються SQL-командою (кнопка «Додати сайт» в адмінці — у планах).

**SQL Editor** → шаблон (замінити назви і розділи під свій сайт) → **Run**:

```sql
insert into public.sites (slug, name, paid_until, config) values (
  'blade',                -- коротке ім'я латиницею, без пробілів
  'BLADE Barbershop',     -- назва, яку бачить клієнт в адмінці
  current_date + 30,      -- підписка: перший місяць
  '{
    "collections": [
      {
        "key": "services",
        "name": "Послуги",
        "fields": [
          {"key": "title", "name": "Назва", "type": "text"},
          {"key": "text",  "name": "Опис",  "type": "textarea"},
          {"key": "price", "name": "Ціна, грн", "type": "text"},
          {"key": "image", "name": "Фото", "type": "image"}
        ]
      }
    ],
    "texts": [
      {"key": "phone",   "name": "Телефон"},
      {"key": "address", "name": "Адреса"}
    ]
  }'
);
```

Після цього сайт одразу з'являється у вашій адмінці — можна наповнювати контент
через кнопки, а потім створити клієнту акаунт (розділ A).

**Словничок полів (для config):**
| type | що це |
|---|---|
| `text` | звичайний рядок |
| `textarea` | багаторядковий текст |
| `checkbox` | так/ні (наприклад, «хіт») |
| `image` | фото із завантаженням |
| `select-collection` | випадний список з іншої колекції (додати `"from": "ключ_колекції"`) |

Основні поля картки — `title`, `text`, `price`, `image` — зберігаються «рідно».
Будь-які додаткові (вага, тривалість, категорія…) — додати `"extra": true` у поле.

## C. Як сайт підключається до бази (хто що робить)

**Робить Claude (кажете «підключи сайт X до бази»):**
1. Пише в коді сайту маленький скрипт: при відкритті сторінки він тягне контент
   із бази (меню, ціни, тексти) і підставляє у верстку
2. Тестує локально, що все підтягується
3. Публікує оновлений сайт

**Робите ви:** нічого технічного 🙂 Тільки дивитесь результат і кажете «ок» або «поправ».

Після підключення ланцюжок працює сам:
**клієнт міняє ціну в адмінці → за хвилину нова ціна на сайті.**

---

## D. Щомісячна рутина підписок

- Клієнт оплатив → адмінка → його картка → **«+30 днів»**
- Не оплатив → після дати закінчення редагування вимикається **само**
  (сайт при цьому працює, вимикається лише адмінка)
- Статуси видно на головному екрані адмінки: зелена пігулка = активна, червона = прострочена

---

## Швидкі відповіді

- **Клієнт забув пароль** → Supabase → Authentication → Users → видалити його → створити
  заново з новим паролем (✅ Auto Confirm) → виконати Крок A3 ще раз
- **Подивитися все меню/контент сайту** → адмінка → Відкрити сайт → вкладки
- **Ключі бази** (для нових деплоїв) → Supabase → Project Settings → API Keys
  (Publishable — публічний; service_role/Secret — НІКОМУ і НІКОЛИ)
- **Фото на сайті клієнта** — два види вкладок:
  - **«Фото сайту»** — готові картки-місця («Номер Стандарт», «Банер коней»…).
    Їх **не видаляють** — лише відкривають і заливають нове фото; назва картки каже,
    куди саме воно стане. Поле «Технічний код місця» не чіпати.
  - **«Галерея»** — вільний список: фото додають, видаляють і переставляють як завгодно.
